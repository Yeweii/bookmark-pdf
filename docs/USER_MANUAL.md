## 1. 快速入门

### 1.1 三步完成书签挂载

```
第一步：选文件
  书签源（TXT/MD） + PDF 文件

第二步：点挂载
  ⚙ 执行挂载 → 等待完成

第三步：检查结果
  进度区显示「完成」，输出 PDF 即含书签导航
```

### 1.2 最简场景：在线获取书签

当没有现成的 TXT/MD 书签文件时：

```
① 输入 SSID（pdfshuwu.com 的书籍编号）
② 点击「🌐 获取书签」
③ 获取成功后选 PDF → 点「⚙ 执行挂载」
④ 挂载完成后自动保存 bookmarks.txt
```

> **无需选择书签源文件**。获取到的书签会直接填入编辑区与预览区。

### 1.3 安装与启动

| 方式 | 操作 |
|------|------|
| **下载分发包**（推荐） | 下载 `BookmarkPDF-macos-arm64.zip`，解压，双击 `BookmarkPDF.app` |
| **pip 安装** | `pip install git+https://github.com/Yeweii/bookmark-pdf.git` 后终端运行 `bookmark-pdf` |
| **源码运行** | `python -m bookmark_pdf` |

> macOS 首次打开 `.app` 时如提示「无法打开」，右键点击.app选择「打开」，或在系统设置中允许运行。

### 3. 解析规则 — Parse Rules

```
┌─ 3. 解析规则 ─────────────────────────────────────────────┐
│  模板: [indent-dot       ▾]  层级模式: [flat       ▾]     │
│  缩进=N 层: [2]          [↻ 重新解析]                    │
│  自定义正则: [________________________________]           │
│  (留空则使用所选模板；正则必须含命名组 ?P<title> 与 ?P<page>) │
└────────────────────────────────────────────────────────────┘
```

- **模板（Template）** — 选择 6 种内置模板之一，或留空使用自定义正则
- **层级模式（Level Mode）** — `flat`（所有同级）/ `indent`（按空格缩进）/ `md_header`（按 `#` 数量）
- **缩进=N 层** — `indent` 模式下，多少空格算一个层级（默认 2）
- **自定义正则** — 需包含 `?P<title>` 与 `?P<page>` 命名组，示例：
  ```
  ^(?P<page>\d+)\s+(?P<title>.+)$
  ```
- **↻ 重新解析** — 强制使用当前规则重新解析书签源文件（忽略缓存）

## 3. 功能场景全解

### 3.1 TXT/MD 文件书签挂载

**适用场景**：已有现成的书签文件（TXT 或 Markdown 格式）。

**操作步骤**：

1. 在「1. 选择文件」点击「书签源」的「浏览…」，选择 `.txt` 或 `.md` 文件
2. 点击「PDF」的「浏览…」，选择目标 PDF
3. 在「3. 解析规则」确认模板选择正确（默认 `flat`，可根据文件格式改为 `indent-dot`、`chapter` 等）
4. 在「4. 预览」确认树形结构与页码正确
5. 在「5. 输出选项」确认输出模式
6. 点击「⚙ 执行挂载」
7. 完成后查看日志确认成功

---

### 3.2 在线获取书签（SSID）

**适用场景**：没有书签文件，但书籍在 pdfshuwu.com 有目录数据。

**操作步骤**：

1. 在「0. 在线获取」输入框填入 SSID（书籍页 URL 中的数字）
2. 点击「🌐 获取书签」
3. 等待获取完成（状态标签更新为书名 + 书签数量）
4. **获取后弹框询问**：「已获取《{书名}》的书签。是否保存为 TXT 文件？」
   - **是** → 弹出文件夹选择 → 保存为 `<书名>_bookmarks.txt`
   - **否** → 继续
5. 在「1. 选择文件」选 PDF（书签源可为空）
6. 点击「⚙ 执行挂载」
7. 挂载完成后**自动保存** `bookmarks.txt` 到 PDF 同目录

**SSID 怎么找？**
打开 pdfshuwu.com，找到目标书籍，URL 形如 `https://pdfshuwu.com/book/13284383` → SSID = `13284383`。

---

### 3.3 从 PDF 读取并编辑书签

**适用场景**：PDF 已有书签，想修改层级、页码或标题后重新挂载。

**操作步骤**：

1. 在「1. 选择文件」只选 PDF（不选书签源）
2. 点击「📥 从 PDF 读取书签」
3. 编辑区自动填入 PDF 现有 outline（`indent-dot` 格式）
4. 如需修改页码：填写「页码 +/-」 Spinbox 后点「应用」
5. 如需批量编辑：点「🔧 工具 ▼」使用工具（见 §3.5）
6. 点「🔄 解析文本」刷新预览树
7. 确认无误后点击「⚙ 执行挂载」

> 读取后可直接在文本区修改标题文字，无需重新解析。

---

### 3.4 粘贴外部文本作为书签源

**适用场景**：从微信、网页、PDF 阅读器等复制了书签文本，直接粘贴使用。

**操作步骤**：

1. 复制外部书签文本（形如 `第一章 引言 1` 或 `1.1 背景 .... 3`）
2. 在「2. 书签文本」编辑区粘贴（Ctrl+V / Cmd+V）
3. 500ms 后自动解析，结果出现在「4. 预览」
4. 在「3. 解析规则」选择与文本格式匹配的模板
5. 检查预览树是否正确，红色条目表示解析异常
6. 选择 PDF → 点「⚙ 执行挂载」

> 如果自动解析结果不对，手动点「🔄 解析文本」强制重新解析。

---

### 3.5 书签文本工具集

点击「🔧 工具 ▼」打开工具弹窗，8 个工具分 4 组：

#### 页码组（Page Number）

| 工具 | 作用 | 示例 |
|------|------|------|
| **归一化（Normalize）** | 从 1 开始按 DFS 顺序重新编号 | `[1, 1, 3]` → `[1, 2, 3]` |
| **首项页码重置（Rebase）** | 将首项页码调整为指定值，后续同步偏移 | 首项 3 → 1：`[3, 7, 10]` → `[1, 5, 8]` |
| **裁剪到最大页（Cap）** | 超过指定页码的设为 `None` | 最大页 10：`11` → `None` |

#### 清理组（Cleanup）

| 工具 | 作用 |
|------|------|
| **去除重复条目（Remove Duplicates）** | `(title, page)` 完全相同的只保留第一个 |
| **移除异常页码条目（Remove Invalid）** | 移除所有 `page=None` 的条目 |

#### 文本组（Text）

| 工具 | 作用 |
|------|------|
| **去除标题首尾空白（Trim Titles）** | `"  第一章  "` → `"第一章"` |

#### 树形组（Tree）

| 工具 | 作用 |
|------|------|
| **展平（Flatten）** | 移除所有层级，所有节点变成顶层 |
| **按页码升序（Sort ↑）** | 按页码从小到大排序，`None` 排末尾 |
| **按页码降序（Sort ↓）** | 按页码从大到小排序，`None` 排末尾 |

**使用方法**：选择工具 → 填写参数（如有）→ 点「执行」→ 工具自动应用到当前节点列表，同步更新文本区与预览树。

---

### 3.6 页码越界的处理方式

**触发条件**：TXT/MD 中的书签页码超过了 PDF 实际页数。

**现象**：点击「⚙ 执行挂载」后弹出对话框：

> 「检测到 {N} 个书签页码超出 PDF 范围（{max_page} 页），是否只挂载范围内的书签（超界节点的子书签将提升到父级）？」

- **是** → 自动过滤越界节点（子节点会提升到父级），重新执行挂载
- **否** → 取消挂载，保持原文件不变

> **举例**：PDF 共 100 页，书签最大页码为 120。选「是」后，页码 101–120 的节点被移除；其中若有子节点（如 `第5章 → 105 → 5.1 小节 110`），`5.1` 会提升到与 `第5章` 同级后挂载。

## 4. 书签格式完全参考

### 4.1 TXT 模板（TXT Templates）

#### flat — 扁平格式

```
1 第一章
2 第二章
3 第三章
```

> 所有条目同级，按行顺序挂载。适合无层级的纯列表书签。

#### indent-dot — 缩进加点格式

```
第一章 .......... 1
  1.1 引言 ..... 2
  1.2 背景 ..... 5
第二章 .......... 10
  2.1 概述 ..... 11
```

> 点号（`.`）连接标题与页码，左侧空格决定层级（默认 2 空格 = 1 级）。

#### chapter — 章节空格格式

```
第一章 引言 1
第二章 背景 5
第三章 总结 20
```

> 标题与页码以空格分隔，适合标题中不含点号的文件。

---

### 4.2 Markdown 模板（MD Templates）

#### md-header-suffix — 标题行内附页码

```markdown
# 概述 .................. 1
## 1.1 背景 ......... 3
## 1.2 相关工作 ..... 8
### 1.2.1 国内研究 .... 10
```

> `#` 数量决定层级，点号连接标题与页码（与 indent-dot 类似，但保留 `#` 层级）。

#### md-header-comment — HTML 注释页码

```markdown
# 概述 <!-- 1 -->
## 背景 <!-- 3 -->
### 1.2.1 国内研究 <!-- 10 -->
```

> 页码写在 HTML 注释 `<!--  -->` 中，标题行保留 Markdown 格式。

#### md-toc-link — TOC 链接格式

```markdown
- [概述](#overview) 1
  - [背景](#background) 3
  - [相关工作](#related) 8
```

> 类似标准 Markdown TOC，`-` 的嵌套决定层级，末尾数字为页码。

---

### 4.3 自定义正则（Custom Regex）

在「3. 解析规则」的「自定义正则」输入框填写。

**必须包含命名组（Named Groups）**：
- `?P<title>` — 标题内容
- `?P<page>` — 页码（数字）

**可选命名组**：
- `?P<indent>` — indent 模式下用于计算层级
- `?P<hashes>` — md_header 模式下用于计算 `#` 数量

#### 示例 1：最简自定义正则

```
^(?P<page>\d+)\s+(?P<title>.+)$
```

匹配：`10 结论` → `title="结论"`, `page="10"`

#### 示例 2：支持括号页码

```
^(?P<title>.+?)\s+\[?(?P<page>\d+)\]?$
```

匹配：`结论 [10]` → `title="结论"`, `page="10"`

#### 示例 3：带缩进自定义正则

```
^(?P<indent>\s*)(?P<title>.+?)\.{3,}(?P<page>\d+)$
```

匹配：`  标题 .... 5` → `indent="  "`, `title="标题"`, `page="5"`
（需配合「层级模式」= indent 使用）

---

### 4.4 层级模式详解（Level Modes）

| 模式 | 说明 | 适用模板 |
|------|------|----------|
| **flat** | 所有条目视为同级 | flat / 自定义正则（无 indent 语义） |
| **indent** | 左侧空格数 ÷ N = 层级 | indent-dot / 自定义含 indent 组 |
| **md_header** | `#` 数量 = 层级 | md-header-suffix / md-header-comment |

---

### 4.5 特殊规则

- **空行** — 自动跳过
- **注释行** — 以 `#`、`//`、`<!--` 开头的行跳过
- **页码非数字** — 解析为 `page=None`，预览中标红，不阻断执行
- **标题缺失** — 抛出 `ParseError`，需修正格式后重试
- **Unicode** — 完全支持中文、emoji、日文等非 ASCII 字符

## 5. 输出选项详解

### 5.1 三种输出模式

| 模式 | 行为 | 输出路径 |
|------|------|----------|
| **新文件（默认）** | 生成新 PDF，原文件不变 | `原文件名_bookmarked.pdf` |
| **原地覆盖** | 直接修改原 PDF | 覆盖原文件（不可恢复） |
| **追加（append）** | 合并到原 outline（保留原有书签） | 覆盖原文件 |

> 追加模式通过 pypdf 的 `mode="append"` 实现，原有 outline 不会丢失。

### 5.2 页码偏移（Page Offset）

- **TXT/MD 页码** — 通常为印刷页码（从 1 开始）
- **PDF 内部页码** — 0-based（从 0 开始）

GUI 默认勾选「页码 -1」，即自动将 TXT 中的 `1` 转换为 PDF 中的 `0`。

**何时去掉勾选**：
- PDF 页码已经是 0-based 时
- 书签源文件中的页码已是 0-based 时

### 5.3 路径预览的作用

「5. 输出选项」中的路径预览标签（`→ /path/to/...`）实时显示实际输出路径。选择 PDF 后、切换模式后、输入框编辑后均会自动刷新。

> **提示**：挂载前务必检查路径预览，确保输出位置符合预期，特别是「原地覆盖」模式下。

## 6. 常见问题与排查

### Q1：启动后界面显示为空白或无响应

**原因**：macOS Gatekeeper 阻止了未签名的 .app。
**解决**：右键点击 `BookmarkPDF.app` → 选择「打开」；或在系统设置 → 隐私与安全性 → 允许运行。

---

### Q2：预览树为空，无任何条目

**排查步骤**：
1. 检查「1. 选择文件」的「书签源」是否已填入
2. 检查「3. 解析规则」的模板是否与文件格式匹配
3. 点击「↻ 重新解析」强制重新解析
4. 检查「2. 书签文本」编辑区是否有内容

---

### Q3：预览中有红色条目

**含义**：红色条目表示 `page=None`（页码解析失败）。
**原因**：
- 原文件中页码处不是数字（如 `第一章 引言` 缺页码）
- 自定义正则在 `page` 组未匹配到数字
**解决**：修正 TXT/MD 文件，或在「2. 书签文本」区手动补全页码。

---

### Q4：挂载时提示「页码越界」

**含义**：TXT/MD 中最大页码超过了 PDF 实际页数。
**解决**：见 §3.6，选「是」跳过越界书签；或修正 TXT/MD 中的页码。

---

### Q5：挂载后 PDF 书签栏看不到书签

**原因 1**：PDF 阅读器未开启书签栏显示。
**解决**：Preview → 查看 → 侧边栏 → 书签；Adobe Reader → 视图 → 面板 → 书签。

**原因 2**：输出的是新文件而不是覆盖文件。
**解决**：检查「5. 输出选项」中路径预览，确认是覆盖模式。

---

### Q6：自定义正则报错或解析结果为空

**排查**：
1. 正则必须包含 `?P<title>` 和 `?P<page>` 两个命名组
2. 使用 [regex101.com](https://regex101.com) 测试正则是否正确
3. 确认「层级模式」设置正确（flat / indent / md_header）

---

### Q7：页码偏移不正确

**判断方法**：
- 如果 TXT 页码是 1-based，PDF 也是 1-based → **去掉**「页码 -1」勾选
- 如果 TXT 页码是 1-based，PDF 是 0-based（大多数情况）→ **保留**「页码 -1」勾选
- 如果 TXT 页码和 PDF 都是 0-based → **去掉**「页码 -1」勾选

**测试方法**：在 PDF 中找一页确认，例如 PDF 第 3 页对应书签「第三章」，若「页码 -1」勾选则挂载为第 2 页。

---

### Q8：保存为新文件后找不到输出文件

**排查**：
1. 检查路径预览标签，确认输出路径
2. 新文件默认与原 PDF 同目录
3. 文件名 = `原文件名_bookmarked.pdf`
4. 如果提示权限错误，换一个非系统目录存放

---

### Q9：在线获取失败

**常见原因**：
- SSID 输入错误（应为纯数字）
- 网络连接问题
- pdfshuwu.com 服务器维护
- 书籍不在 pdfshuwu.com 目录中（不支持）

**解决**：检查 SSID 是否正确，尝试刷新网络后重试。

### 7.2 Python API

作为模块导入使用：

```python
from pathlib import Path
from bookmark_pdf.parser import Parser, BookmarkNode
from bookmark_pdf.bookmark import mount_bookmarks

# 1. 解析 TXT/MD
rule = Parser.BUILTIN_RULES["indent-dot"]
nodes = Parser(rule).parse_file(Path("bookmarks.txt"))

# 2. 挂载到 PDF
mount_bookmarks(
    pdf_path=Path("input.pdf"),
    nodes=nodes,
    output_path=Path("output.pdf"),
    mode="replace",
    page_offset=-1,          # 1-based → 0-based
)

# 3. 从 PDF 读取现有书签
from bookmark_pdf.bookmark import read_bookmarks
nodes = read_bookmarks(Path("input.pdf"))

# 4. BookmarkNode 序列化为 indent-dot 文本
from bookmark_pdf.parser import to_indent_dot
text = to_indent_dot(nodes)

# 5. 书签文本变换（纯函数）
from bookmark_pdf.transforms import shift_pages, normalize_pages, remove_duplicates
shifted = shift_pages(nodes, offset=+3)
normalized = normalize_pages(nodes, start=1)
deduped = remove_duplicates(nodes)
```

### 7.3 BookmarkNode 数据结构

```python
from dataclasses import dataclass

@dataclass
class BookmarkNode:
    title: str              # 非空字符串
    page: int | None       # 1-based；None 表示异常
    line_no: int            # 源文件行号（用于错误定位）
    children: list         # 子节点列表
```

### 7.4 异常类型

| 异常 | 触发场景 | GUI 处理 |
|------|----------|----------|
| `ParseError` | 正则错误、标题缺失 | 弹框提示，解析中断 |
| `PageOutOfRangeError` | 页码超 PDF 上限 | 弹框询问是否过滤重试 |
| `PdfReadError` | PDF 不存在或损坏 | 弹框提示 |
| `PermissionError` | 无写入权限 | 弹框提示 |
